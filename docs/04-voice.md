# Principles

**Status:** Approved  
**Version:** 1.0

## The Four Pillars

Every meaningful feature or design decision must strengthen at least one pillar.

### Wonder

The universe should never become ordinary.

Vesper preserves a sense of scale, discovery, and emotional connection without turning scientific information into spectacle.

### Precision

Measured, not guessed.

Accuracy, source honesty, clear units, and transparent uncertainty are part of the visual and product experience.

### Calm

Silence is a feature.

The product uses restraint, generous space, low-noise motion, and disciplined notifications. It never competes with the sky.

### Discovery

Every session should reveal something worth remembering or sharing.

The user should leave with a clear answer to at least one question:

- What can I see?
- When can I see it?
- Where should I look?
- Why does it matter?

## Operational Principles

### Respect the Night

Darkness is not empty. It is the environment in which the product operates.

### Respect Attention

Every interruption must repay the user with useful sky information.

### Gold Is Earned

Gold marks significance. It is not a default button color or decorative treatment.

### Motion Explains

Animation must communicate change, direction, state, hierarchy, or continuity.

### Precision Feels Beautiful

Alignment, rhythm, typography, and data presentation should feel measured.

### Point Before Explaining

Lead with the object, glyph, direction, time, or diagram. Supporting text follows.

### Honest About Clouds

When conditions are poor, say so plainly.

### The Telescope Test

Remove the logo, labels, and brand name. The interface should still feel designed for astronomical observation rather than adapted from a generic app template.

## Review Checklist

- Does this respect the night?
- Does this respect attention?
- Which pillar does it strengthen?
- Does it help the user observe?
- Is it accurate and honest?
- Is anything present only to look impressive?

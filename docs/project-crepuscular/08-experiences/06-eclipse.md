# Eclipse

**Status:** Draft for review  
**Version:** 0.1

## Purpose

The eclipse experience is Vesper's v1 ceremonial centerpiece.

It must make the 12 August 2026 solar eclipse locally understandable, safe, and actionable.

## Core Principle

> Wonder never outranks eye safety.

## Entry States

The experience changes with time.

### Long Before

Focus on:

- local visibility
- countdown
- obscuration
- planning
- safety education
- location

### Seven Days Before

Add:

- cloud outlook
- observation preparation
- reminder status
- share card

### Day Before

Prioritize:

- exact local times
- location confirmation
- forecast confidence
- equipment and safety
- backup plan

### Event Day

Prioritize:

- countdown
- contact timeline
- current weather
- Sun altitude
- safety
- live status where supported

### During the Eclipse

The interface becomes a focused instrument.

Show:

- current phase
- time to next contact
- local obscuration progress
- safety instruction
- weather state
- minimal secondary content

### After

Show:

- event complete
- observation log
- share result
- next relevant eclipse
- no forced rating or upgrade request

## Local Data

Required:

- partial beginning
- maximum
- partial end
- obscuration
- Sun altitude at maximum
- local time zone
- location confidence

All values use one canonical data source within the app.

## Safety

Safety content is:

- always free
- always reachable
- visible before event-day actions
- direct
- non-poetic
- not dismissible permanently before the event

Core message:

> Never look directly at the Sun without certified eclipse eye protection.

Do not dilute safety with brand language.

## Ceremonial Visuals

Allowed:

- horizon light
- solar geometry
- contact arcs
- stronger spatial motion
- controlled gold and ember
- atmospheric scale

Not allowed:

- effects that obscure times
- fantasy eclipse colors
- rapid flashing
- dramatic countdown motion that creates stress
- safety text over unstable imagery

## Timeline

The eclipse timeline should show:

- current time
- contact points
- maximum
- progress
- local daylight context

It must have a text alternative.

## Weather

Show:

- cloud cover at eclipse hour
- forecast confidence
- update time
- nearby time variation where useful
- honest uncertainty

Do not promise visibility from an unstable forecast.

## Alerts

Default milestones:

- D−7
- D−1
- T−1 h
- T−10 min

Each alert must explain what to do next.

## Share Card

Required:

- location
- date
- local contact times
- obscuration
- Vesper identity
- safety reminder where appropriate

The share card must use canonical local data and remain readable without app context.

## Accessibility

- countdown is not the only time representation
- every phase has a text label
- Reduced Motion offers immediate states
- safety is screen-reader prioritized
- large text does not obscure contact times
- color is not the only phase indicator

## Anti-Patterns

Do not:

- place safety behind Pro
- use a paywall on event day
- exaggerate forecast certainty
- show different times in alerts and screen
- use spectacle that competes with action
- ask for a review immediately after maximum

# Observation Log

**Status:** Draft for review  
**Version:** 0.1

## Purpose

The Observation Log helps users remember what they saw and become more confident observers.

Logging remains free.

## Core Principle

> Recording the sky should take less time than observing it.

## Quick Log

A quick log should require only:

- object
- date and time
- location
- confirmation

Optional:

- note
- photo
- equipment
- conditions
- rating or visibility

## Entry Points

Allow logging from:

- Object Detail
- Tonight
- alert destination
- Eclipse
- Observation Log home

## Log Form

Recommended order:

1. object
2. seen time
3. location
4. short note
5. photo
6. equipment
7. conditions

Pre-fill known context.

Do not make users re-enter data Vesper already knows.

## Log Home

Show:

- recent observations
- object
- date
- location
- thumbnail where present
- simple search or filter
- empty-state invitation

## Empty State

Direction:

> Your first observation begins outside. When you see something, record it here.

Keep the action practical rather than literary.

## Detail

An entry may show:

- observation data
- note
- photo
- conditions snapshot
- object link
- edit
- delete
- share

## Pro

The core log remains free forever.

Pro may add:

- export
- statistics
- advanced equipment analysis
- long-term patterns

Do not restrict access to existing entries after entitlement changes.

## Offline

Logging must work offline.

- save locally
- sync later where applicable
- show sync state
- prevent duplicates
- never lose a completed entry because the network failed

## Success

Use:

- immediate confirmation
- subtle haptic
- short message
- return to observation context where appropriate

No confetti. No review prompt.

## Accessibility

- photo is optional
- all fields have labels
- date and time are editable
- saved state is announced
- entries remain readable at large text
- destructive deletion requires confirmation

## Anti-Patterns

Do not:

- require an account
- require a photo
- force sharing
- make the form long by default
- paywall basic logging
- lose offline entries
- interrupt the user with celebration

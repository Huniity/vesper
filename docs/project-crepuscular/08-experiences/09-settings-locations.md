# Settings and Locations

**Status:** Draft for review  
**Version:** 0.1

## Purpose

Settings should be quiet, transparent, and easy to leave.

They are a solid instrument surface, not a branded showcase.

## Structure

Recommended groups:

1. Location
2. Notifications
3. Appearance
4. Language
5. Pro
6. Privacy and data
7. About and sources

## Location

Show:

- active location
- approximate or manual status
- location freshness
- saved locations
- free or Pro limit
- add, edit, remove, and set primary actions

Precise coordinates do not need to be exposed by default.

## Multiple Locations

Free:

- one location

Pro:

- unlimited saved locations

The current location remains usable without an account.

When a limit is reached:

- explain the value
- preserve the existing location
- offer Pro without urgency
- never block editing or deleting

## Notifications

Each topic shows:

- enabled state
- plain-language description
- current threshold
- quiet-hours behavior
- Pro options where applicable

Do not place every advanced threshold on the first level.

## Appearance

Options:

- Standard dark
- Red night mode
- system or manual selection where appropriate
- Reduced Motion information may link to system settings

Red night mode should be quick to reach from observation contexts, not only buried in Settings.

## Language

Supported at launch:

- Português (Portugal)
- Português (Brasil)
- English
- Español

Language changes must update the whole interface without requiring account creation.

## Privacy and Data

Show:

- anonymous-first explanation
- location handling
- notification token handling
- delete my data
- privacy policy
- data export where available

Deletion:

- requires confirmation
- explains consequence
- completes without dark patterns
- returns the app to a clean local state

## About

Include:

- version
- data sources
- licenses
- image credits
- MET Norway attribution
- support
- privacy and deletion links

## Visual Treatment

Use:

- solid surfaces
- scan-friendly rows
- restrained separators
- no background animation
- no decorative planet imagery behind controls
- clear section headings

## Accessibility

- rows expand with large text
- switches include meaning
- destructive actions are explicit
- language names remain in their native form
- external links are announced
- focus order follows the visual grouping

## Anti-Patterns

Do not:

- hide privacy controls
- place upgrade prompts in every group
- use glass settings cards
- require an account to change language
- obscure location status
- make deletion difficult
